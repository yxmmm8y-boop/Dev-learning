# Отчёт о выполнении тестового задания DevOps

## 1. Описание выполненных шагов

В рамках выполнения задания был реализован процесс развёртывания микросервиса на виртуальной машине с использованием Ansible.

### 1.1 Развёртывание виртуальной машины

На хостовой системе Fedora была настроена среда виртуализации с использованием KVM, libvirt и virt-manager.
Создана виртуальная машина с операционной системой Rocky Linux 9.

В процессе установки:

* создан пользователь с правами администратора;
* включено сетевое подключение;
* настроен hostname.

---

### 1.2 Настройка сети

После установки было выявлено отсутствие доступа в интернет из виртуальной машины.

В ходе диагностики:

* проверен IP-адрес (команда `ip a`);
* проверен маршрут (команда `ip route`);
* проверена доступность шлюза (ping 192.168.122.1);
* проверена сеть libvirt на хосте (`virsh net-list --all`).

Было установлено, что проблема связана с отсутствием корректного NAT.

На хостовой системе Fedora были настроены правила:

```bash
sudo iptables -t nat -F
sudo iptables -F
sudo iptables -t nat -A POSTROUTING -s 192.168.122.0/24 -o wlp4s0 -j MASQUERADE
sudo iptables -A FORWARD -i virbr0 -o wlp4s0 -j ACCEPT
sudo iptables -A FORWARD -i wlp4s0 -o virbr0 -m state --state RELATED,ESTABLISHED -j ACCEPT
```

После этого доступ в интернет был восстановлен.

---

### 1.3 Настройка SSH

В виртуальной машине установлен SSH-сервер:

```bash
sudo dnf install -y openssh-server
sudo systemctl enable --now sshd
```

На хосте создан SSH-ключ:

```bash
ssh-keygen -t ed25519
```

Ключ добавлен на ВМ:

```bash
ssh-copy-id yxmmmy@192.168.122.209
```

---

### 1.4 Настройка Ansible

На Fedora установлен Ansible:

```bash
sudo dnf install -y ansible
```

Создан файл inventory:

```ini
[servers]
rocky ansible_host=192.168.122.209 ansible_user=yxmmmy
```

Проверка подключения:

```bash
ansible -i inventory.ini servers -m ping
```

---

### 1.5 Разработка микросервиса

Разработан микросервис на Python с использованием встроенного HTTP-сервера.

Функционал:

* прослушивание порта 8080;
* обработка endpoint `/metrics`;
* возврат метрики в формате Prometheus.

---

### 1.6 Развёртывание через Ansible

Создан playbook, выполняющий:

* установку Python;
* создание директории `/opt/microservice`;
* копирование файла сервиса;
* запуск сервиса.

Запуск:

```bash
ansible-playbook -i inventory.ini playbook.yml -K
```

---

### 1.7 Настройка firewall

Сервис был недоступен извне. Причина — закрытый порт.

Решение:

```bash
sudo firewall-cmd --add-port=8080/tcp --permanent
sudo firewall-cmd --reload
```

### 1.8 Исправление ошибок

В процессе выполнения были устранены ошибки:

* отсутствие интернет-доступа;
* некорректный NAT;

## 3. Команды, использованные в Ansible

Основные задачи playbook:

```yaml
- name: Install python
  dnf:
    name: python3
    state: present

- name: Create directory
  file:
    path: /opt/microservice
    state: directory

- name: Copy service
  copy:
    src: service/app.py
    dest: /opt/microservice/app.py

- name: Run service
  shell: nohup python3 /opt/microservice/app.py &
```
