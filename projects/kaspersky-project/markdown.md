# Тестовое задание DevOps

## Общая информация

В рамках выполнения тестового задания был реализован процесс развёртывания микросервиса на виртуальной машине с использованием системы управления конфигурациями Ansible.

Целью работы являлось:

* развернуть виртуальную машину на базе Linux;
* настроить сетевое взаимодействие;
* реализовать микросервис, экспортирующий метрики;
* автоматизировать процесс развёртывания;
* обеспечить доступ к сервису по HTTP.

---

## Используемые технологии

* Fedora Linux (хостовая система)
* Rocky Linux 9 (виртуальная машина)
* KVM / libvirt / virt-manager
* Ansible
* Python 3

---

## Развёртывание виртуальной машины

На хостовой системе Fedora была настроена среда виртуализации на базе KVM и libvirt. С помощью virt-manager была создана виртуальная машина с операционной системой Rocky Linux 9.

В процессе установки был создан пользователь с административными правами и настроено сетевое подключение.

---

## Настройка сети

После установки было выявлено, что виртуальная машина не имеет доступа в интернет.

В ходе диагностики было установлено:

* виртуальная машина получает IP-адрес;
* шлюз доступен;
* маршрутизация настроена корректно;
* проблема находится на стороне хостовой системы.

Причиной являлось отсутствие корректного NAT для подсети виртуальных машин.

Для решения проблемы на хосте Fedora были вручную настроены правила iptables:

```bash
sudo iptables -t nat -F
sudo iptables -F
sudo iptables -t nat -A POSTROUTING -s 192.168.122.0/24 -o wlp4s0 -j MASQUERADE
sudo iptables -A FORWARD -i virbr0 -o wlp4s0 -j ACCEPT
sudo iptables -A FORWARD -i wlp4s0 -o virbr0 -m state --state RELATED,ESTABLISHED -j ACCEPT
```

После применения данных правил доступ в интернет из виртуальной машины был восстановлен.

---

## Настройка SSH-доступа

В виртуальной машине был установлен и запущен SSH-сервер:

```bash
sudo dnf install -y openssh-server
sudo systemctl enable --now sshd
```

На хостовой системе был создан SSH-ключ и добавлен на виртуальную машину с помощью команды:

```bash
ssh-copy-id yxmmmy@192.168.122.209
```

Это позволило обеспечить подключение без ввода пароля и использовать Ansible.

---

## Настройка Ansible

На Fedora был установлен Ansible.
Создан файл inventory:

```ini
[servers]
rocky ansible_host=192.168.122.209 ansible_user=yxmmmy
```

Проверка подключения выполнена с помощью:

```bash
ansible -i inventory.ini servers -m ping
```

---

## Реализация микросервиса

Был разработан микросервис на Python, использующий встроенный HTTP-сервер.

Сервис:

* работает на порту 8080;
* обрабатывает endpoint `/metrics`;
* возвращает метрику в формате Prometheus.

Пример ответа:

```text
host_type{host="rocky-vm"} 1
```

---

## Развёртывание сервиса с помощью Ansible

Был создан playbook, выполняющий следующие действия:

* установка Python 3;
* создание директории `/opt/microservice`;
* копирование файла сервиса;
* запуск сервиса.

Playbook запускается командой:

```bash
ansible-playbook -i inventory.ini playbook.yml -K
```

---

## Настройка доступа к сервису

После развёртывания сервис был недоступен извне из-за настроек firewalld.

Для решения проблемы был открыт порт 8080:

```bash
sudo firewall-cmd --add-port=8080/tcp --permanent
sudo firewall-cmd --reload
```

---

## Устранение ошибок

В процессе выполнения были устранены следующие проблемы:

* отсутствие интернет-доступа в ВМ;
* некорректная настройка NAT;
* отсутствие SSH-ключей;
* ошибка выполнения Ansible без sudo-пароля;
* блокировка порта firewalld;
* ошибка `Empty reply from server`, вызванная некорректным формированием HTTP-ответа.

---

## Проверка работы сервиса

Проверка выполняется командой:

```bash
curl http://192.168.122.209:8080/metrics
```

Ожидаемый результат:

```text
host_type{host="rocky-vm"} 1
```

Docker в рамках данного решения не использовался.
