# Git теория
Git  - Система контроля версий 

## Задачи:
 1) Сохранять историю изменений файлов
 2) видеть, что было изменено 
 3) откатываться назад
 4) синхронизировать с GitHub
 5) уверенность, что при "поломке" можно будет вернуться к рабочей версии 

## Основные понятия:
 *Repository* - папка проекта, в который Git отслеживает изменения.

 *Commit* - снимок состояния проекта в определнный момент времени.

 *Sraging Area* - область подготовки файлов перед commit.
 
 *Branch* - отдельная линия ("ветка") разработки.

 *Remote repository* - Удаленный репозиторий (GitHub).



## Структура Git
 *Папка* на компьютере = проект
 
 *Git* = журнал изменений этой папки
 
 *GitHub* = сайт, куда ты выкладываешь этот журнал и сами файлы

 *Working Directory* — файлы проекта

 *Staging Area* — подготовка

 *Repository* — история коммитов

## Принцип работы Git
1) Изменение файлов
2) Добавление их в staging (git add)
3) Создание commit (git commit)
4) Отправка на GitHub (git push)
