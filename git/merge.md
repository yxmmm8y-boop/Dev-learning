# Git Merge

## Теория
Merge — это объединение изменений из одной ветки в другую.

---

## Пример

```bash
git switch main
git merge feature
```

---

## Конфликты

Возникают, если изменения в файлах пересекаются.

Пример:

```text
<<<<<<< HEAD
мой код
=======
другой код
>>>>>>> feature
```

---

## Как решить конфликт

1. Открыть файл
2. Удалить лишние строки
3. Оставить нужный код
4. Сохранить

```bash
git add .
git commit -m "Resolve conflict"
```

---

## Fast-forward merge

Если ветки не расходились, Git просто перемещает указатель.

---

## Когда использовать merge

 - после завершения работы в ветке
 - перед отправкой изменений
