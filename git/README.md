#Git_теория
Git  - Система контроля версий 
##Задачи:
 1) Сохранять историю изменений файлов
 2) видеть, что было изменено 
 3) откатываться назад
 4) синхронизировать с GitHub
 5) уверенность, что при "поломке" можно будет вернуться к рабочей версии 

##Принцип работы
Папка на компьютере = проект
Git = журнал изменений этой папки
GitHub = сайт, куда ты выкладываешь этот журнал и сами файлы

=> Работа локально в Linux -> Git отслеживет изменения -> отправляется на GitHub

##Команды


###Базовые команды:

####Git status - показывает текущее состояние репозитория:
 - измененные файлы
 - новые файлы
 - добавленные в commit (зафиксированный «снимок» (snapshot) состояния всех файлов вашего проекта в определенный момент времени)
 - текущую ветку 

####Git add - добавляет изменения в staging area (подготовка к commit)
"git add ." - добавляет изменения в текущей папке и ниже
"git add -A" - добавляет ВСЕ во всем репозитории 

####Git commit -m  - сохраняет изменения в истории Git и добавляет massage(сообщение коммита):
 - берет все из staging area
 - записывает в историю

####Git push - отправляет коммиты на GitHub
Без этой команды изменения только у тебя на компьютере

####Git pull - скачивает изменения с GitHub
 -  использовать перед началом работы
 - подходит, если общий проект или работа с разных устройств


###Команды для просмотра:

####Git log - показывает историю коммитов

####Git diff - показывает изменения в файлах до commit

####Git show - последний commit с изменениями


###Команды для работы с файлами:

####git rm - удаляет файл из Git, и с диска

####git mv - переименовывает файл

####git restore - отменяет изменения файла


###Работа с Ветками

####git branch - показывает ветки

####git branch feature - создает ветку

####git switch/chtckout feature - перейти в ветку

####git merge feature - объединяет ветки 


###Связь с GITHUB

####Git remote -v - показывает куда подключен репозиторий 

####git clone - скачивает репозиторий с GitHub

####git pull origin main - явно отправляет в ветку main

#### git pull origin main - явно скачивает из main


###Откат и Ошиибки

####Git reset - удаляет последний комит но оставляет изменения 

####Git reset --hard - удаляет все безвозвратно 

####Git checkout файл - откатывает файл к последнему комиту
